---
title: Rozwiązywanie problemów
description: Typowe problemy i sposoby ich rozwiązywania.
---

Aby debugować problemy z HanuBees, zacznij od sprawdzenia dzienników i danych lokalnych przechowywanych na dysku.

---

## Dzienniki

Pliki logów są zapisywane w:

- **macOS/Linux**: `~/.local/share/HanuBees/log/`
- **Windows**: Naciśnij `WIN+R` i wklej `%USERPROFILE%\.local\share\HanuBees\log`

Nazwy plików dziennika zawierają znaczniki czasu (np. `2025-01-09T123456.log`) i przechowywanych jest 10 ostatnich plików dziennika.

Możesz ustawić poziom dziennika za pomocą opcji wiersza poleceń `--log-level`, aby uzyskać bardziej szczegółowe informacje debugowania. Na przykład `HanuBees --log-level DEBUG`.

---

## Przechowywanie danych

HanuBees przechowuje dane sesji i inne dane aplikacji na dysku pod adresem:

- **macOS/Linux**: `~/.local/share/HanuBees/`
- **Windows**: Naciśnij `WIN+R` i wklej `%USERPROFILE%\.local\share\HanuBees`

Ten katalog zawiera:

- `auth.json` - Authentication data like API keys, OAuth tokens
- `log/` - Application logs
- `project/` — Dane specyficzne dla projektu, takie jak dane sesji i komunikatów
  - Jeśli projekt znajduje się w repozytorium Git, jest on przechowywany w `./<project-slug>/storage/`
  - If it is not a Git repo, it is stored in `./global/storage/`

---

## Aplikacja desktopowa

HanuBees Desktop uruchamia w tle lokalny serwer HanuBees (sidecar `HanuBees-cli`). Większość problemów jest spowodowana nieprawidłowo działającą wtyczką, uszkodzoną pamięcią podręczną lub złymi ustawieniami serwera.

### Szybkie sprawdzenie

- Całkowicie zakończ i uruchom ponownie aplikację.
- Jeśli aplikacja wyświetli ekran błędu, kliknij **Uruchom ponownie** i skopiuj szczegóły błędu.
- Tylko macOS: menu `HanuBees` -> **Załaduj ponownie przeglądarkę internetową** (pomaga, jeśli interfejs użytkownika jest pusty/zawieszony).

---

### Wyłączanie wtyczek

Jeśli aplikacja komputerowa ulega awarii podczas uruchamiania, zawiesza się lub zachowuje się dziwnie, zacznij od wyłączenia wtyczek.

#### Sprawdź konfigurację globalną

Otwórz globalny plik konfiguracyjny i poszukaj klucza `plugin`.

- **macOS/Linux**: `~/.config/HanuBees/HanuBees.jsonc` (lub `~/.config/HanuBees/HanuBees.json`)
- **macOS/Linux** (starsze instalacje): `~/.local/share/HanuBees/HanuBees.jsonc`
- **Windows**: Naciśnij `WIN+R` i wklej `%USERPROFILE%\.config\HanuBees\HanuBees.jsonc`

Jeśli masz skonfigurowane wtyczki, tymczasowo je wyłącz, usuwając klucz lub ustawiając go na pustą tablicę:

```jsonc
{
  "$schema": "https://hanubees.com/config.json",
  "plugin": [],
}
```

#### Sprawdź katalogi wtyczek

HanuBees może także ładować lokalne wtyczki z dysku. Tymczasowo usuń je (lub zmień nazwę folderu) i uruchom ponownie aplikację komputerową:

- **Wtyczki globalne**
  - **macOS/Linux**: `~/.config/HanuBees/plugins/`
  - **Windows**: Naciśnij `WIN+R` i wklej `%USERPROFILE%\.config\HanuBees\plugins`
- **Wtyczki projektowe** (tylko jeśli używasz konfiguracji dla poszczególnych projektów)
  - `<your-project>/.HanuBees/plugins/`

Jeśli aplikacja ponownie zacznie działać, włączaj ponownie wtyczki pojedynczo, aby dowiedzieć się, która powoduje problem.

---

### Wyczyść pamięć podręczną

Jeśli wyłączenie wtyczek nie pomoże (lub instalacja wtyczki utknęła), wyczyść pamięć podręczną, aby HanuBees mógł ją odbudować.

1. Całkowicie zamknij HanuBees Desktop.
2. Usuń katalog pamięci podręcznej:

- **macOS**: Finder -> `Cmd+Shift+G` -> paste `~/.cache/HanuBees`
- **Linux**: usuń `~/.cache/HanuBees` (lub uruchom `rm -rf ~/.cache/HanuBees`)
- **Windows**: Naciśnij `WIN+R` i wklej `%USERPROFILE%\.cache\HanuBees`

3. Uruchom ponownie HanuBees Desktop.

---

### Napraw problemy z połączeniem z serwerem

HanuBees Desktop może uruchomić własny serwer lokalny (domyślnie) lub połączyć się ze skonfigurowanym adresem URL serwera.

Jeśli zobaczysz okno dialogowe **„Połączenie nie powiodło się”** (lub aplikacja nigdy nie wychodzi poza ekran powitalny), sprawdź, czy jest niestandardowy adres URL serwera.

#### Wyczyść domyślny adres URL serwera na komputerze

Na ekranie głównym kliknij nazwę serwera (z kropką stanu), aby otworzyć selektor serwerów. W sekcji **Serwer domyślny** kliknij **Wyczyść**.

#### Usuń `server.port` / `server.hostname` ze swojej konfiguracji

Jeśli Twój `HanuBees.json(c)` zawiera sekcję `server`, tymczasowo usuń ją i uruchom ponownie aplikację komputerową.

#### Sprawdź zmienne środowiskowe

Jeśli w swoim środowisku masz ustawiony `HanuBees_PORT`, aplikacja komputerowa spróbuje użyć tego portu dla serwera lokalnego.

- usuń `HanuBees_PORT` (lub wybierz wolny port) i uruchom ponownie.

---

### Linux: Problemy z Wayland / X11

W systemie Linux, niektóre konfiguracje Wayland mogą powodować puste okna lub błędy kompozytora.

- Jeśli korzystasz z Wayland, a aplikacja jest pusta/ ulega awarii, spróbuj uruchomić ją za pomocą `OC_ALLOW_WAYLAND=1`.
- Jeśli to pogorszy sprawę, usuń go i zamiast tego spróbuj uruchomić w sesji X11.

---

### Windows: Środowisko uruchomieniowe WebView2

W systemie Windows HanuBees Desktop wymaga Microsoft Edge **WebView2 Runtime**. Jeśli aplikacja otwiera się w pustym oknie lub nie uruchamia się, zainstaluj/zaktualizuj WebView2 i spróbuj ponownie.

---

### Windows: Ogólne problemy z wydajnością

Jeśli doświadczasz niskiej wydajności, problemów z dostępem do plików lub problemów z terminalem w systemie Windows, spróbuj użyć [WSL (podsystem Windows dla systemu Linux) (./windows-wsl). WSL zapewnia środowisko Linux, które płynniej współpracuje z funkcjami HanuBees.

---

### Brak powiadomień

HanuBees Desktop pokazuje powiadomienia systemowe tylko wtedy, gdy:

- powiadomienia są włączone dla HanuBees w ustawieniach systemu operacyjnego, oraz
- okno aplikacji nie jest aktywne.

---

### Resetowanie pamięci aplikacji

Jeśli aplikacja nie uruchamia się i nie możesz wyczyścić ustawień w interfejsie użytkownika, zresetuj zapisany stan aplikacji komputerowej.

1. Zamknij całkowicie HanuBees Desktop.
2. Znajdź i usuń te pliki (znajdują się w katalogu danych aplikacji HanuBees Desktop):

- `HanuBees.settings.dat` (domyślny adres URL serwera na komputerze stacjonarnym)
- `HanuBees.global.dat` i `HanuBees.workspace.*.dat` (stan interfejsu użytkownika, taki jak najnowsze serwery/projekty)

Aby szybko znaleźć katalog:

- **macOS**: Finder -> `Cmd+Shift+G` -> `~/Library/Application Support` (następnie wyszukaj nazwy plików powyżej)
- **Linux**: wyszukaj powyższe nazwy plików pod `~/.local/share`
- **Windows**: Naciśnij `WIN+R` -> `%APPDATA%` (następnie wyszukaj nazwy plików powyżej)

## Uzyskiwanie pomocy

Jeśli masz problemy z HanuBees:

1. **Zgłoś problem na GitHub**

   Najlepszym sposobem zgłaszania błędów lub zgłaszania żądań funkcji jest skorzystanie z naszego repozytorium GitHub:

   [**github.com/varsansri/hanubees-coding-agent/issues**](https://github.com/varsansri/hanubees-coding-agent/issues)

   Przed utworzeniem nowego problemu przeszukaj istniejące i sprawdź, czy Twój problem nie został już zgłoszony.

2. **Dołącz do naszego Discorda**

   Aby uzyskać pomoc w czasie rzeczywistym i dyskusję społeczności, dołącz do naszego serwera Discord:

   [**hanubees.com/discord**](https://hanubees.com/discord)

## Typowe problemy

Oto kilka typowych problemów i sposobów ich rozwiązania.

---

### HanuBees nie uruchamia się

1. Sprawdź dzienniki pod kątem komunikatów o błędach
2. Spróbuj uruchomić z `--print-logs`, aby zobaczyć dane wyjściowe w terminalu
3. Upewnij się, że masz najnowszą wersję z `HanuBees upgrade`

---

### Problemy z uwierzytelnianiem

1. Spróbuj ponownie uwierzytelnić się za pomocą polecenia `/connect` w TUI
2. Sprawdź, czy klucze API są ważne
3. Upewnij się, że Twoja sieć umożliwia połączenia z interfejsem API dostawcy

---

### Model niedostępny

1. Sprawdź, czy dokonałeś uwierzytelnienia u dostawcy
2. Sprawdź, czy nazwa modelu w konfiguracji jest poprawna
3. Niektóre modele mogą wymagać określonego dostępu lub subskrypcji

Jeśli napotkasz `ProviderModelNotFoundError`, najprawdopodobniej
błędnie odwołujesz się gdzieś do modelu.
Modele powinny być wskazywane w ten sposób: `<providerId>/<modelId>`

Przykłady:

- `openai/gpt-4.1`
- `openrouter/google/gemini-2.5-flash`
- `HanuBees/kimi-k2`

Aby dowiedzieć się, do jakich modeli masz dostęp, uruchom `HanuBees models`

---

### ProviderInitError

Jeśli napotkasz błąd ProviderInitError, prawdopodobnie masz nieprawidłową lub uszkodzoną konfigurację.

Aby rozwiązać ten problem:

1. Najpierw sprawdź, czy Twój dostawca jest prawidłowo skonfigurowany, postępując zgodnie z [przewodnikiem dostawców](/docs/providers)
2. Jeśli problem będzie się powtarzał, spróbuj wyczyścić zapisaną konfigurację:

   ```bash
   rm -rf ~/.local/share/HanuBees
   ```

   W systemie Windows naciśnij `WIN+R` i usuń: `%USERPROFILE%\.local\share\HanuBees`

3. Ponownie uwierzytelnij się u swojego dostawcy za pomocą polecenia `/connect` w TUI.

---

### Błędy AI_APICallError i problemy z pakietami dostawców

Jeśli napotkasz błędy wywołań API, może to wynikać z nieaktualnych pakietów dostawców. HanuBees dynamicznie instaluje pakiety dostawców (OpenAI, Anthropic, Google itp.) w razie potrzeby i przechowuje je lokalnie w pamięci podręcznej.

Aby rozwiązać problemy z pakietem dostawcy:

1. Wyczyść pamięć podręczną pakietu dostawcy:

   ```bash
   rm -rf ~/.cache/HanuBees
   ```

   W systemie Windows naciśnij `WIN+R` i usuń: `%USERPROFILE%\.cache\HanuBees`

2. Uruchom ponownie kod HanuBees, aby ponownie zainstalować najnowsze pakiety dostawców

Zmusi to HanuBees do pobrania najnowszych wersji pakietów dostawców, co często rozwiązuje problemy ze zgodnością z parametrami modelu i zmianami API.

---

### Kopiowanie/wklejanie nie działa na Linuxie

Aby funkcja kopiowania/wklejania działała, użytkownicy systemu Linux muszą mieć zainstalowane jedno z następujących narzędzi schowka:

**Dla systemów X11:**

```bash
apt install -y xclip
# or
apt install -y xsel
```

**Dla systemów Wayland:**

```bash
apt install -y wl-clipboard
```

**Dla środowisk headless:**

```bash
apt install -y xvfb
# and run:
Xvfb :99 -screen 0 1024x768x24 > /dev/null 2>&1 &
export DISPLAY=:99.0
```

HanuBees wykryje, czy używasz Waylanda i wolisz `wl-clipboard`, w przeciwnym razie spróbuje znaleźć narzędzia schowka w kolejności: `xclip` i `xsel`.
