HanuBees CLI จะเริ่มต้น [TUI](/docs/tui) เป็นค่าเริ่มต้นเมื่อเรียกใช้โดยไม่มีอาร์กิวเมนต์

```bash
HanuBees
```

นอกจากนี้ยังรับคำสั่งและแฟล็ก ซึ่งช่วยให้คุณใช้ HanuBees ผ่านโปรแกรมได้

```bash
HanuBees run "Explain how closures work in JavaScript"
```

---

### tui

เริ่ม TUI เพื่อใช้ HanuBees แบบโต้ตอบ

```bash
HanuBees [project]
```

#### แฟล็ก

| แฟล็ก                                    | สั้น | คำอธิบาย                                                   |
| ---------------------------------------- | ---- | ---------------------------------------------------------- |
| <nobr><code>{"--continue"}</code></nobr> | `-c` | ดำเนินการต่อจากเซสชันล่าสุด                                |
| <nobr><code>{"--session"}</code></nobr>  | `-s` | ID เซสชันเพื่อดำเนินการต่อ                                 |
| <nobr><code>{"--fork"}</code></nobr>     |      | แยกเซสชันเมื่อทำต่อ (ใช้กับ `--continue` หรือ `--session`) |
| <nobr><code>{"--prompt"}</code></nobr>   |      | พรอมต์เริ่มต้นที่จะใช้                                     |
| <nobr><code>{"--model"}</code></nobr>    | `-m` | โมเดลที่จะใช้ในรูปแบบ provider/model                       |
| <nobr><code>{"--agent"}</code></nobr>    |      | เอเจนต์ที่จะใช้                                            |
| <nobr><code>{"--port"}</code></nobr>     |      | พอร์ตที่จะฟัง                                              |
| <nobr><code>{"--hostname"}</code></nobr> |      | ชื่อโฮสต์ที่จะฟัง                                          |

### attach

เชื่อมต่อกับเซิร์ฟเวอร์แบ็กเอนด์ HanuBees ที่กำลังทำงานอยู่ ซึ่งเริ่มต้นผ่านคำสั่ง `serve` หรือ `web`

```bash
HanuBees attach [url]
```

วิธีนี้ช่วยให้คุณใช้ TUI กับแบ็กเอนด์ HanuBees ระยะไกลได้ ตัวอย่าง:

```bash
# Start the backend server for web/mobile access
HanuBees web --port 4096 --hostname 0.0.0.0

# In another terminal, attach the TUI to the running backend
HanuBees attach http://10.20.30.40:4096
```

| แฟล็ก                                    | สั้น | คำอธิบาย                                                                                    |
| ---------------------------------------- | ---- | ------------------------------------------------------------------------------------------- |
| <nobr><code>{"--dir"}</code></nobr>      |      | ไดเร็กทอรีการทำงานเพื่อเริ่มต้น TUI ใน                                                      |
| <nobr><code>{"--continue"}</code></nobr> | `-c` | ดำเนินการต่อจากเซสชันล่าสุด                                                                 |
| <nobr><code>{"--session"}</code></nobr>  | `-s` | ID เซสชันเพื่อดำเนินการต่อ                                                                  |
| <nobr><code>{"--fork"}</code></nobr>     |      | แยกเซสชันเมื่อดำเนินการต่อ (ใช้กับ `--continue` หรือ `--session`)                           |
| <nobr><code>{"--password"}</code></nobr> | `-p` | รหัสผ่านการยืนยันตัวตนพื้นฐาน (ค่าเริ่มต้นคือ `HanuBees_SERVER_PASSWORD`)                   |
| <nobr><code>{"--username"}</code></nobr> | `-u` | ชื่อผู้ใช้การยืนยันตัวตนพื้นฐาน (ค่าเริ่มต้นคือ `HanuBees_SERVER_USERNAME` หรือ `HanuBees`) |

```bash
HanuBees auth [command]
```

#### login

HanuBees ได้รับการกำหนดค่าไว้ล่วงหน้าด้วยรายชื่อผู้ให้บริการทั่วไปที่ [Models.dev](https://models.dev) ดังนั้นคุณเพียงแค่ต้องใส่คีย์ API สำหรับผู้ให้บริการที่คุณต้องการใช้ ข้อมูลนี้จะถูกเก็บไว้ที่ `~/.local/share/HanuBees/auth.json`

```bash
HanuBees auth login
```

เมื่อ HanuBees เริ่มต้น ระบบจะโหลดข้อมูลรับรองจากไฟล์นี้ ดังนั้นคุณจึงไม่ต้องกังวลกับการตั้งค่าตัวแปรสภาพแวดล้อมหรือไฟล์ `.env` ในโปรเจ็กต์ของคุณ

---

#### list

แสดงรายการข้อมูลรับรองการตรวจสอบสิทธิ์ทั้งหมดที่กำหนดค่าไว้

```bash
HanuBees auth list
```

หรือใช้นามแฝงสั้น

```bash
HanuBees auth ls
```

---

#### logout

```bash
HanuBees auth logout
```

#### add

เพิ่มเซิร์ฟเวอร์ MCP ลงในการกำหนดค่าของคุณ

```bash
HanuBees mcp add
```

คำสั่งนี้จะแนะนำคุณผ่านวิซาร์ดเพื่อกำหนดค่าเซิร์ฟเวอร์ MCP ใหม่

---

#### list

แสดงรายชื่อเซิร์ฟเวอร์ MCP ทั้งหมดที่กำหนดค่าไว้และสถานะ

```bash
HanuBees mcp list
```

หรือใช้นามแฝงสั้น

```bash
HanuBees mcp ls
```

---

#### auth

ยืนยันตัวตนกับเซิร์ฟเวอร์ MCP ที่ต้องการ OAuth

```bash
HanuBees mcp auth [name]
```

หากไม่ระบุชื่อเซิร์ฟเวอร์ คุณจะได้รับแจ้งให้เลือกจากเซิร์ฟเวอร์ที่รองรับ OAuth ที่พร้อมใช้งาน

คุณยังสามารถแสดงรายการสถานะการตรวจสอบสิทธิ์ OAuth ได้:

```bash
HanuBees mcp auth list
```

หรือใช้นามแฝงสั้น

```bash
HanuBees mcp auth ls
```

---

#### logout

```bash
HanuBees mcp logout [name]
```

### models

แสดงรายการโมเดลที่พร้อมใช้งานทั้งหมดจากผู้ให้บริการที่กำหนดค่าไว้

```bash
HanuBees models [provider]
```

คำสั่งนี้จะแสดง ID โมเดลทั้งหมดในรูปแบบ `provider/model`

คุณสามารถใช้ชื่อโมเดลเหล่านี้ในการกำหนดค่า [config](/docs/config/) ของคุณ

คุณยังสามารถส่งผ่านรหัสผู้ให้บริการเพื่อกรองโมเดลตามผู้ให้บริการรายนั้นได้

```bash
HanuBees models anthropic
```

#### แฟล็ก

| แฟล็ก                                   | คำอธิบาย                                               |
| --------------------------------------- | ------------------------------------------------------ |
| <nobr><code>{"--refresh"}</code></nobr> | รีเฟรชแคชโมเดลจาก models.dev                           |
| <nobr><code>{"--verbose"}</code></nobr> | แสดงรายละเอียดโมเดลเพิ่มเติม (รวมข้อมูลเมตาเช่นต้นทุน) |
| <nobr><code>{"--json"}</code></nobr>    | แสดงผลลัพธ์เป็น JSON                                   |

ใช้แฟล็ก `--refresh` เพื่ออัปเดตรายการโมเดลที่แคชไว้ มีประโยชน์เมื่อมีการเพิ่มโมเดลใหม่ให้กับผู้ให้บริการและคุณต้องการเห็นใน HanuBees

```bash
HanuBees models --refresh
```

### run

รัน HanuBees โดยไม่ต้องใช้ TUI โดยส่งพรอมต์โดยตรง

```bash
HanuBees run [message..]
```

มีประโยชน์สำหรับการทำงานอัตโนมัติหรือเมื่อต้องการคำตอบอย่างรวดเร็วโดยไม่ต้องเปิด TUI ตัวอย่าง:

```bash "HanuBees run"
HanuBees run Explain the use of context in Go
```

คุณยังสามารถแนบไปกับเซิร์ฟเวอร์ `HanuBees serve` ที่ทำงานอยู่ หรือใช้เซิร์ฟเวอร์ MCP แบบ headless:

```bash
# Start a headless server in one terminal
HanuBees serve

# In another terminal, run commands that attach to it
HanuBees run --attach http://localhost:4096 "Explain async/await in JavaScript"
```

### serve

เริ่มต้นเซิร์ฟเวอร์ HanuBees headless ที่ให้บริการ API HTTP ดู [เอกสารเซิร์ฟเวอร์](/docs/server) สำหรับรายละเอียด API เต็มรูปแบบ

```bash
HanuBees serve
```

คำสั่งนี้จะเริ่มต้นเซิร์ฟเวอร์ HTTP ให้ API เข้าถึงฟังก์ชันการทำงานของ HanuBees ได้โดยไม่ต้องมี TUI นอกจากนี้ยังรองรับการตรวจสอบสิทธิ์พื้นฐาน HTTP (ชื่อผู้ใช้เริ่มต้นคือ `HanuBees` และรหัสผ่านระบุโดย `HanuBees_SERVER_PASSWORD`)

#### แฟล็ก

| แฟล็ก                                    | คำอธิบาย                               |
| ---------------------------------------- | -------------------------------------- |
| <nobr><code>{"--port"}</code></nobr>     | พอร์ตที่จะฟัง                          |
| <nobr><code>{"--hostname"}</code></nobr> | ชื่อโฮสต์ที่จะฟัง                      |
| <nobr><code>{"--mdns"}</code></nobr>     | เปิดใช้งานการค้นหา mDNS                |
| <nobr><code>{"--cors"}</code></nobr>     | ต้นกำเนิดเพิ่มเติมที่อนุญาตสำหรับ CORS |

| <nobr><code>{"--cors"}</code></nobr>     | ต้นกำเนิดเพิ่มเติมที่อนุญาตสำหรับ CORS |

HanuBees acp

#### แฟล็ก

| <nobr><code>{"--port"}</code></nobr>     | พอร์ตที่จะฟัง      |
| <nobr><code>{"--hostname"}</code></nobr> | ชื่อโฮสต์ที่จะฟัง  |
