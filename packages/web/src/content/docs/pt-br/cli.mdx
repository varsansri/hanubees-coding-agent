A CLI do HanuBees, por padrão, inicia o [TUI](/docs/tui) quando executada sem argumentos.

```bash
HanuBees
```

Mas também aceita comandos conforme documentado nesta página. Isso permite que você interaja com o HanuBees programaticamente.

```bash
HanuBees run "Explain how closures work in JavaScript"
```

---

### tui

Inicie a interface de usuário do terminal do HanuBees.

```bash
HanuBees [project]
```

#### Opções

| Flag                                     | Curto | Descrição                                                                  |
| ---------------------------------------- | ----- | -------------------------------------------------------------------------- |
| <nobr><code>{"--continue"}</code></nobr> | `-c`  | Continue a última sessão                                                   |
| <nobr><code>{"--session"}</code></nobr>  | `-s`  | ID da sessão para continuar                                                |
| <nobr><code>{"--fork"}</code></nobr>     |       | Criar um fork da sessão ao continuar (use com `--continue` ou `--session`) |
| <nobr><code>{"--prompt"}</code></nobr>   |       | Prompt a ser usado                                                         |
| <nobr><code>{"--model"}</code></nobr>    | `-m`  | Modelo a ser usado na forma de provider/model                              |
| <nobr><code>{"--agent"}</code></nobr>    |       | Agente a ser usado                                                         |
| <nobr><code>{"--port"}</code></nobr>     |       | Porta para escutar                                                         |
| <nobr><code>{"--hostname"}</code></nobr> |       | Nome do host para escutar                                                  |

## Comandos

A CLI do HanuBees também possui os seguintes comandos.

---

### agent

Gerencie agentes para o HanuBees.

```bash
HanuBees agent [command]
```

---

### attach

Anexe um terminal a um servidor backend do HanuBees já em execução, iniciado via comandos `serve` ou `web`.

```bash
HanuBees attach [url]
```

Isso permite usar o TUI com um backend HanuBees remoto. Por exemplo:

```bash
# Inicie o servidor backend para acesso web/mobile
HanuBees web --port 4096 --hostname 0.0.0.0

# Em outro terminal, anexe o TUI ao backend em execução
HanuBees attach http://10.20.30.40:4096
```

#### Opções

| Flag                                     | Curto | Descrição                                                                         |
| ---------------------------------------- | ----- | --------------------------------------------------------------------------------- |
| <nobr><code>{"--dir"}</code></nobr>      |       | Diretório de trabalho para iniciar o TUI                                          |
| <nobr><code>{"--continue"}</code></nobr> | `-c`  | Continuar a última sessão                                                         |
| <nobr><code>{"--session"}</code></nobr>  | `-s`  | ID da sessão para continuar                                                       |
| <nobr><code>{"--fork"}</code></nobr>     |       | Bifurcar a sessão ao continuar (use com `--continue` ou `--session`)              |
| <nobr><code>{"--password"}</code></nobr> | `-p`  | Senha de autenticação básica (padrão: `HanuBees_SERVER_PASSWORD`)                 |
| <nobr><code>{"--username"}</code></nobr> | `-u`  | Usuário de autenticação básica (padrão: `HanuBees_SERVER_USERNAME` ou `HanuBees`) |

#### create

Crie um novo agente com configuração personalizada.

```bash
HanuBees agent create
```

Este comando irá guiá-lo na criação de um novo agente com um prompt de sistema personalizado e configuração de ferramentas.

#### list

Liste todos os agentes disponíveis.

```bash
HanuBees agent list
```

### auth

Comando para gerenciar credenciais e login para provedores.

```bash
HanuBees auth [command]
```

---

#### login

O HanuBees é alimentado pela lista de provedores em [Models.dev](https://models.dev), então você pode usar `HanuBees auth login` para configurar chaves de API para qualquer provedor que você gostaria de usar. Isso é armazenado em `~/.local/share/HanuBees/auth.json`.

```bash
HanuBees auth login
```

Quando o HanuBees é iniciado, ele carrega os provedores do arquivo de credenciais. E se houver chaves definidas em seus ambientes ou em um arquivo `.env` em seu projeto.

---

#### list

Lista todos os provedores autenticados conforme armazenado no arquivo de credenciais.

```bash
HanuBees auth list
```

Ou a versão curta.

```bash
HanuBees auth ls
```

---

#### logout

Desconecta você de um provedor limpando-o do arquivo de credenciais.

```bash
HanuBees auth logout
```

### github

Gerencie o agente do GitHub para automação de repositórios.

```bash
HanuBees github [command]
```

---

#### install

Instale o agente do GitHub em seu repositório.

```bash
HanuBees github install
```

Isso configura o fluxo de trabalho necessário do GitHub Actions e o guia pelo processo de configuração. [Saiba mais](/docs/github).

---


### mcp

Gerencie servidores do Model Context Protocol.

```bash
HanuBees mcp [command]
```

---

#### add

Adicione um servidor MCP à sua configuração.

```bash
HanuBees mcp add
```

Este comando irá guiá-lo na adição de um servidor MCP local ou remoto.

---

#### list

Liste todos os servidores MCP configurados e seu status de conexão.

```bash
HanuBees mcp list
```

Ou use a versão curta.

```bash
HanuBees mcp ls
```

---

#### auth

Autentique-se com um servidor MCP habilitado para OAuth.

```bash
HanuBees mcp auth [name]
```

Se você não fornecer um nome de servidor, será solicitado que você selecione entre os servidores disponíveis habilitados para OAuth.

Você também pode listar servidores habilitados para OAuth e seu status de autenticação.

```bash
HanuBees mcp auth list
```

Ou use a versão curta.

```bash
HanuBees mcp auth ls
```

---

#### logout

Remova credenciais OAuth para um servidor MCP.

```bash
HanuBees mcp logout [name]
```

---

#### debug

Depure problemas de conexão OAuth para um servidor MCP.

```bash
HanuBees mcp debug <name>
```

### models

Liste todos os modelos disponíveis dos provedores configurados.

```bash
HanuBees models [provider]
```

Este comando exibe todos os modelos disponíveis entre seus provedores configurados no formato `provider/model`.

Isso é útil para descobrir o nome exato do modelo a ser usado em [sua configuração](/docs/config/).

Você pode opcionalmente passar um ID de provedor para filtrar modelos por esse provedor.

```bash
HanuBees models anthropic
```

#### Opções

| Flag                                    | Descrição                                                             |
| --------------------------------------- | --------------------------------------------------------------------- |
| <nobr><code>{"--refresh"}</code></nobr> | Atualiza o cache de modelos a partir do models.dev                    |
| <nobr><code>{"--verbose"}</code></nobr> | Use uma saída de modelo mais detalhada (inclui metadados como custos) |

Use a flag `--refresh` para atualizar a lista de modelos em cache. Isso é útil quando novos modelos foram adicionados a um provedor e você deseja vê-los no HanuBees.

```bash
HanuBees models --refresh
```

### run

Execute o HanuBees em modo não interativo passando um prompt diretamente.

```bash
HanuBees run [message..]
```

Isso é útil para scripts, automação ou quando você deseja uma resposta rápida sem iniciar o TUI completo. Por exemplo.

```bash "HanuBees run"
HanuBees run Explique o uso de context em Go
```

Você também pode se anexar a uma instância em execução do `HanuBees serve` para evitar tempos de inicialização a frio do servidor MCP em cada execução:

```bash
# Inicie um servidor sem cabeça em um terminal
HanuBees serve

# Em outro terminal, execute comandos que se anexam a ele
HanuBees run --attach http://localhost:4096 "Explique async/await em JavaScript"
```

#### Opções

| Flag                                     | Curto | Descrição                                                                         |
| ---------------------------------------- | ----- | --------------------------------------------------------------------------------- |
| <nobr><code>{"--command"}</code></nobr>  |       | O comando a ser executado, use mensagem para argumentos                           |
| <nobr><code>{"--continue"}</code></nobr> | `-c`  | Continue a última sessão                                                          |
| <nobr><code>{"--session"}</code></nobr>  | `-s`  | ID da sessão para continuar                                                       |
| <nobr><code>{"--fork"}</code></nobr>     |       | Criar um fork da sessão ao continuar (use com `--continue` ou `--session`)        |
| <nobr><code>{"--share"}</code></nobr>    |       | Compartilhe a sessão                                                              |
| <nobr><code>{"--model"}</code></nobr>    | `-m`  | Modelo a ser usado na forma de provider/model                                     |
| <nobr><code>{"--agent"}</code></nobr>    |       | Agente a ser usado                                                                |
| <nobr><code>{"--file"}</code></nobr>     | `-f`  | Arquivo(s) a serem anexados à mensagem                                            |
| <nobr><code>{"--format"}</code></nobr>   |       | Formato: padrão (formatado) ou json (eventos JSON brutos)                         |
| <nobr><code>{"--title"}</code></nobr>    |       | Título para a sessão (usa o prompt truncado se nenhum valor for fornecido)        |
| <nobr><code>{"--attach"}</code></nobr>   |       | Anexe a um servidor HanuBees em execução (por exemplo, http://localhost:4096)     |
| <nobr><code>{"--password"}</code></nobr> | `-p`  | Senha de autenticação básica (padrão: `HanuBees_SERVER_PASSWORD`)                 |
| <nobr><code>{"--username"}</code></nobr> | `-u`  | Usuário de autenticação básica (padrão: `HanuBees_SERVER_USERNAME` ou `HanuBees`) |
| <nobr><code>{"--dir"}</code></nobr>      |       | Diretório de execução, ou caminho no servidor remoto ao anexar                    |
| <nobr><code>{"--variant"}</code></nobr>  |       | Variante do modelo (nível de raciocínio específico do provedor)                   |
| <nobr><code>{"--thinking"}</code></nobr> |       | Mostrar blocos de pensamento                                                      |
| <nobr><code>{"--port"}</code></nobr>     |       | Porta para o servidor local (padrão para porta aleatória)                         |

### serve

Inicie um servidor HanuBees sem cabeça para acesso à API. Confira a [documentação do servidor](/docs/server) para a interface HTTP completa.

```bash
HanuBees serve
```

Isso inicia um servidor HTTP que fornece acesso à funcionalidade do HanuBees sem a interface TUI. Defina `HanuBees_SERVER_PASSWORD` para habilitar a autenticação básica HTTP (o nome de usuário padrão é `HanuBees`).

#### Opções

| Flag                                     | Descrição                                             |
| ---------------------------------------- | ----------------------------------------------------- |
| <nobr><code>{"--port"}</code></nobr>     | Porta para escutar                                    |
| <nobr><code>{"--hostname"}</code></nobr> | Nome do host para escutar                             |
| <nobr><code>{"--mdns"}</code></nobr>     | Habilitar descoberta mDNS                             |
| <nobr><code>{"--cors"}</code></nobr>     | Origem(ns) de navegador adicionais para permitir CORS |

### stats

Mostre o uso de tokens e estatísticas de custo para suas sessões do HanuBees.

```bash
HanuBees stats
```

#### Opções

| Flag                                    | Descrição                                                                                            |
| --------------------------------------- | ---------------------------------------------------------------------------------------------------- |
| <nobr><code>{"--days"}</code></nobr>    | Mostre estatísticas dos últimos N dias (todo o tempo)                                                |
| <nobr><code>{"--tools"}</code></nobr>   | Número de ferramentas a serem mostradas (todas)                                                      |
| <nobr><code>{"--models"}</code></nobr>  | Mostre a divisão do uso de modelos (oculto por padrão). Passe um número para mostrar os N principais |
| <nobr><code>{"--project"}</code></nobr> | Filtrar por projeto (todos os projetos, string vazia: projeto atual)                                 |

### export

Exporte dados da sessão como JSON.

```bash
HanuBees export [sessionID]
```

Se você não fornecer um ID de sessão, será solicitado que você selecione entre as sessões disponíveis.

### import

Importe dados da sessão de um arquivo JSON ou URL de compartilhamento do HanuBees.

```bash
HanuBees import <file>
```

Você pode importar de um arquivo local ou de uma URL de compartilhamento do HanuBees.

```bash
HanuBees import session.json
HanuBees import https://opncd.ai/s/abc123
```

### uninstall


| ------------------------------------------- | ----- | ---------------------------------------- |

| <nobr><code>{"--force"}</code></nobr>       | `-f`  | Pular prompts de confirmação             |

### upgrade

Atualiza o HanuBees para a versão mais recente ou uma versão específica.

```bash
HanuBees upgrade [target]
```

Para atualizar para a versão mais recente.

```bash
HanuBees upgrade
```

Para atualizar para uma versão específica.

```bash
HanuBees upgrade v0.1.48
```

#### Opções

| Flag                                   | Curto | Descrição                                                        |
| -------------------------------------- | ----- | ---------------------------------------------------------------- |
| <nobr><code>{"--method"}</code></nobr> | `-m`  | O método de instalação que foi usado; curl, npm, pnpm, bun, brew |

## Variáveis de ambiente

O HanuBees pode ser configurado usando variáveis de ambiente.

| Variável                              | Tipo    | Descrição                                                             |
| ------------------------------------- | ------- | --------------------------------------------------------------------- |
| `HanuBees_AUTO_SHARE`                 | boolean | Compartilhar sessões automaticamente                                  |
| `HanuBees_GIT_BASH_PATH`              | string  | Caminho para o executável do Git Bash no Windows                      |
| `HanuBees_CONFIG`                     | string  | Caminho para o arquivo de configuração                                |
| `HanuBees_CONFIG_DIR`                 | string  | Caminho para o diretório de configuração                              |
| `HanuBees_CONFIG_CONTENT`             | string  | Conteúdo de configuração json inline                                  |
| `HanuBees_DISABLE_AUTOUPDATE`         | boolean | Desabilitar verificações de atualização automática                    |
| `HanuBees_DISABLE_PRUNE`              | boolean | Desabilitar a poda de dados antigos                                   |
| `HanuBees_DISABLE_TERMINAL_TITLE`     | boolean | Desabilitar atualizações automáticas do título do terminal            |
| `HanuBees_PERMISSION`                 | string  | Configuração de permissões json inline                                |
| `HanuBees_DISABLE_DEFAULT_PLUGINS`    | boolean | Desabilitar plugins padrão                                            |
| `HanuBees_DISABLE_LSP_DOWNLOAD`       | boolean | Desabilitar downloads automáticos do servidor LSP                     |
| `HanuBees_ENABLE_EXPERIMENTAL_MODELS` | boolean | Habilitar modelos experimentais                                       |
| `HanuBees_DISABLE_AUTOCOMPACT`        | boolean | Desabilitar compactação automática de contexto                        |
| `HanuBees_DISABLE_CLAUDE_CODE`        | boolean | Desabilitar leitura de `.claude` (prompt + habilidades)               |
| `HanuBees_DISABLE_CLAUDE_CODE_PROMPT` | boolean | Desabilitar leitura de `~/.claude/CLAUDE.md`                          |
| `HanuBees_DISABLE_CLAUDE_CODE_SKILLS` | boolean | Desabilitar carregamento de `.claude/skills`                          |
| `HanuBees_DISABLE_MODELS_FETCH`       | boolean | Desabilitar busca de modelos de fontes remotas                        |
| `HanuBees_FAKE_VCS`                   | string  | Provedor VCS falso para fins de teste                                 |
| `HanuBees_CLIENT`                     | string  | Identificador do cliente (padrão é `cli`)                             |
| `HanuBees_ENABLE_EXA`                 | boolean | Habilitar ferramentas de busca web Exa                                |
| `HanuBees_SERVER_PASSWORD`            | string  | Habilitar autenticação básica para `serve`/`web`                      |
| `HanuBees_SERVER_USERNAME`            | string  | Substituir nome de usuário de autenticação básica (padrão `HanuBees`) |
| `HanuBees_MODELS_URL`                 | string  | URL personalizada para buscar configuração de modelos                 |
