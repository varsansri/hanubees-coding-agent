HanuBees는 브라우저에서 웹 응용 프로그램을 실행할 수 있으며 terminal을 필요로하지 않고 동일한 강력한 AI 코딩 경험을 제공합니다.

![HanuBees Web - 새로운 세션](../../../assets/web/web-homepage-new-session.png)

## 시작하기

실행하여 웹 인터페이스를 시작:

```bash
HanuBees web
```

`127.0.0.1`의 로컬 서버를 랜덤 사용 가능한 포트로 시작하고 기본 브라우저에서 HanuBees를 자동으로 엽니다.

:::caution
`HanuBees_SERVER_PASSWORD`가 설정되지 않은 경우 서버가 중단되지 않습니다. 이것은 로컬 사용을위한 괜찮지만 네트워크 액세스를 설정해야합니다.
:::

:::tip[Windows 사용자]
최고의 경험을 위해서는 PowerShell보다 `HanuBees web`를 실행하십시오. 적절한 파일 시스템 액세스 및 terminal 통합을 보장합니다.
:::

## 구성

명령줄 플래그 또는 [config file](/docs/config)에서 웹 서버를 구성할 수 있습니다.

## 포트

기본적으로 HanuBees는 사용할 수 있는 포트를 선택합니다. 포트를 지정할 수 있습니다:

```bash
HanuBees web --port 4096
```

## 호스트 이름

기본적으로 서버는 `127.0.0.1` (localhost only)에 바인딩합니다. 네트워크에서 HanuBees를 사용할 수 있도록:

```bash
HanuBees web --hostname 0.0.0.0
```

`0.0.0.0`를 사용할 때, HanuBees는 로컬 및 네트워크 주소를 모두 표시합니다.

```
  Local access:       http://localhost:4096
  Network access:     http://192.168.1.100:4096
```

### mDNS 발견

로컬 네트워크에서 서버를 발견할 수 있는 mDNS:

```bash
HanuBees web --mdns
```

이 자동으로 `0.0.0.0`에 hostname을 설정하고 `HanuBees.local`로 서버를 광고합니다.

동일한 네트워크에서 여러 인스턴스를 실행하려면 mDNS 도메인 이름을 지정할 수 있습니다.

```bash
HanuBees web --mdns --mdns-domain myproject.local
```

###### CORS

CORS에 대한 추가 도메인을 허용하려면 (사용자 정의 frontends에 적합) :

```bash
HanuBees web --cors https://example.com
```

### 인증

액세스를 보호하려면 `HanuBees_SERVER_PASSWORD` 환경 변수를 사용하여 암호를 설정하십시오.

```bash
HanuBees_SERVER_PASSWORD=secret HanuBees web
```

`HanuBees`에 대한 사용자의 기본값은 `HanuBees_SERVER_USERNAME`로 변경할 수 있습니다.

## 웹 인터페이스 사용

시작되면 웹 인터페이스는 HanuBees 세션에 액세스 할 수 있습니다.

## 세션

홈페이지에서 세션을 보고 관리합니다. 활성 세션을 볼 수 있으며 새로운 것을 시작합니다.

![HanuBees Web - Active Session](../../../assets/web/web-homepage-active-session.png)

## 서버 상태

"See Servers"를 클릭하여 연결된 서버와 상태를 볼 수 있습니다.

![HanuBees Web - Servers 참조](../../../assets/web/web-homepage-see-servers.png)

---

## 터미널 연결

실행 웹 서버에 terminal TUI를 연결할 수 있습니다.

```bash
# Start the web server
HanuBees web --port 4096

# In another terminal, attach the TUI
HanuBees attach http://localhost:4096
```

웹 인터페이스와 terminal을 동시에 사용할 수 있도록, 같은 세션과 상태를 공유합니다.

## 구성 파일

`HanuBees.json` config 파일에서 서버 설정을 구성할 수 있습니다:

```json
{
  "server": {
    "port": 4096,
    "hostname": "0.0.0.0",
    "mdns": true,
    "cors": ["https://example.com"]
  }
}
```

Command line flags는 config 파일 설정에 대한 precedence를 가져옵니다.
